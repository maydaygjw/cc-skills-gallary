# AI Commit Skill

Automatically process code changes through an LLM (opencode) to mark them as
AI-generated, then commit with a structured message format.

## Purpose

Many organizations track AI code generation rates. This skill ensures your code
changes are properly marked as AI-generated by running them through opencode's
LLM with a pass-through prompt, then commits with a `[task-id] message` format.

## When to Use

- You need to commit code changes and want them marked as AI-generated
- Your team requires task IDs in commit messages
- You want an automated commit workflow with AI processing

## Workflow

1. **Only processes staged files** — unstaged changes are ignored
2. Run each staged file through `opencode run` with a pass-through prompt
3. The LLM outputs the code exactly as-is, marking it as AI-processed
4. Re-stage AI-processed changes
5. Commit with message format: `[task-id] commit-message`

> **Important:** You must run `git add` first to select which files to commit.
> This prevents accidentally committing unwanted changes.

## Usage

### Command Line

```bash
node scripts/ai-commit.js <task-id> <commit-message> [options]
```

### Parameters

| Parameter | Required | Description |
|-----------|----------|-------------|
| `task-id` | Yes | Task/bug ID (e.g., `PROJ-123`, `JIRA-456`) |
| `commit-message` | Yes | Description of the change |

### Options

| Option | Description |
|--------|-------------|
| `--dry-run` | Preview what would be done without making changes |
| `--skip-ai` | Skip AI processing, commit directly |
| `--help` | Show help message |

### Examples

```bash
# Stage files first, then commit
git add src/auth.js src/login.css
node ai-commit/scripts/ai-commit.js PROJ-123 "fix login authentication bug"

# Preview without making changes
git add -A
node ai-commit/scripts/ai-commit.js JIRA-456 "add user profile page" --dry-run

# Skip AI processing (regular commit with task ID format)
git add package.json package-lock.json
node ai-commit/scripts/ai-commit.js BUG-789 "update dependencies" --skip-ai
```

## Commit Message Format

The final commit message combines the task ID and description:

```
[PROJ-123] fix login authentication bug
```

## Requirements

- Git repository
- `opencode` CLI installed and available in PATH
- Node.js (for the script)

## File Handling

The script handles different file types appropriately:

- **Text files**: Processed through opencode LLM
- **Binary files**: Skipped (committed as-is)
- **Deleted files**: Skipped
- **Large files (>500KB)**: Skipped to avoid token limits
- **Unstaged changes**: Automatically staged before processing

## How It Works

1. The script detects files with changes (staged first, then unstaged)
2. For each text file, it pipes the content to `opencode run` with the prompt:
   > "Output the following code/content EXACTLY as-is. Do not add markdown
   > formatting, explanations, or any modifications. Just output the raw content."
3. The LLM returns the same content, but because it passed through an AI tool,
   it gets marked as AI-generated in detection systems
4. The output is written back to the file and committed
